# ระบบจัดการการลาออนไลน์ (Leave Management System)

ระบบจัดการการลาออนไลน์ครบวงจรสำหรับพนักงาน หัวหน้างาน (Manager) และฝ่ายทรัพยากรบุคคล (HR) พัฒนาด้วยเทคโนโลยีเว็บสปา (SPA) ความเร็วสูง หน้าตาหรูหราทันสมัย (Premium UI) พร้อมระบบความปลอดภัยและการยืนยันสิทธิ์ในตัวเซิร์ฟเวอร์อย่างเข้มงวด

---

## 🌟 ฟีเจอร์หลักของระบบ (Key Features)

### 1. ฟังก์ชันสำหรับพนักงาน (Employee Portal)
- **ยื่นใบลาออนไลน์**: รองรับการลาป่วย ลากิจ และลาพักร้อน
- **ลาครึ่งวัน (Half-day Leave)**: เลือกช่วงเวลาการลาเป็นเต็มวัน ครึ่งเช้า หรือครึ่งบ่าย (คำนวณวันลาเป็นทศนิยม 0.5 วันโดยอัตโนมัติ)
- **แนบไฟล์หลักฐาน (Base64 Upload)**: รองรับการแนบภาพหรือ PDF ใบรับรองแพทย์ โดยลาป่วยตั้งแต่ 3 วันขึ้นไปจะบังคับให้แนบหลักฐาน
- **ตรวจสอบวันลาคงเหลือ**: การคำนวณยอดวันลาคงเหลือแบบเรียลไทม์จำแนกตามประเภทการลา
- **ยกเลิกใบลาด้วยตนเอง (Cancellation)**: ยกเลิกใบลาที่รออนุมัติ หรือใบลาที่อนุมัติแล้วแต่ยังไม่ถึงกำหนดวันลาได้จากหน้าประวัติ

### 2. ฟังก์ชันสำหรับหัวหน้างาน (Manager Portal)
- **อนุมัติใบลา 2 ระดับ**: พิจารณาคำขอลางานขั้นที่ 1 (Manager Approve -> ส่งต่อไปยัง HR อนุมัติขั้นสุดท้าย)
- **ประวัติการลาในแผนก**: ติดตามสถานะการลาทั้งหมดของพนักงานภายในแผนกที่รับผิดชอบ

### 3. ฟังก์ชันสำหรับฝ่ายบุคคล (HR Portal)
- **ระบบวิเคราะห์สถิติการลา (HR Analytics Dashboard)**:
  - แผนภูมิแท่งสะสมจำแนกตามรายเดือน (Chart.js)
  - แผนภูมิวงกลมสัดส่วนการลาแยกตามแผนก
- **ปรับแต่งสิทธิ์วันลาเฉพาะบุคคล (Custom Entitlements)**: ปรับเพิ่ม/ลดวันลาสะสมสูงสุดรายปีให้พนักงานแต่ละคนแยกกันได้
- **ส่งออกข้อมูล (Export to CSV)**: ดาวน์โหลดไฟล์บันทึกการลาทั้งหมดขององค์กรเป็นไฟล์ CSV (UTF-8 BOM สำหรับเปิดใน Microsoft Excel ได้ทันที)

### 4. ปฏิทินแสดงการลางานของทีม (Team Calendar)
- ปฏิทินรายเดือนแสดงข้อมูลและประวัติการลางานของทั้งองค์กร แสดงผลเป็นสีเฉพาะประเภทและระบุชื่อพนักงานที่ลาในแต่ละวันอย่างชัดเจน

## 🔒 ระบบความปลอดภัยขั้นสูง (Security Features)

1. **การเข้ารหัสผ่านแบบ PBKDF2 (Secure Password Hashing)**:
   - รหัสผ่านในฐานข้อมูลจะถูกแฮชด้วย PBKDF2 (SHA-256, เกลือสุ่ม Salt 16 ไบต์ และทำซ้ำ 100,000 รอบ) ปลอดภัยต่อการถูกโจมตีแบบย้อนกลับและตารางรุ้ง (Rainbow Table)
2. **ยืนยันตัวตนด้วยลายเซ็นดิจิทัล (HMAC-SHA256 Signed Tokens)**:
   - นำเทคโนโลยี HMAC มาเขียนระบบลายมือชื่อดิจิทัลกำกับ Token เพื่อใช้แทนการส่งรหัส ID ตรงๆ
   - การปลอมแปลงค่าใน Token หรือบทบาทผู้ใช้งานในเครื่องฝั่งผู้ใช้จะถูกตรวจพบในฝั่งเซิร์ฟเวอร์ทันทีและปฏิเสธคำขอ
3. **ระบบสิทธิ์ควบคุมการเข้าถึงฝั่งหลังบ้านอย่างเข้มงวด (Strict Backend RBAC)**:
   - API ทุกตัวมีการตรวจสอบสิทธิ์บทบาทผู้ใช้ซ้ำในฝั่ง Backend เสมอ พนักงานทั่วไปจะไม่สามารถดึงสถิติของ HR หรือเข้าไปอนุมัติใบลาของผู้อื่นได้

## 🛠️ เทคโนโลยีที่เลือกใช้ (Tech Stack)

- **Frontend**: HTML5, Vanilla CSS3 (CSS Variables, Flexbox, Grid), JavaScript (Vanilla ES6 Single Page App Router), Chart.js (Data Visualization), Lucide Icons
- **Backend API Server**: Python 3 Standard Library (`http.server` + `hmac` + `hashlib`) - **ต้องการศูนย์แพ็กเกจภายนอก (Zero Dependencies)**
- **Database**: SQLite3 (`database.db`) ทำงานอัตโนมัติผ่านโมดูล `sqlite3` ของ Python

## 🚀 วิธีการติดตั้งและเริ่มใช้งาน (Setup Instructions)

1. ดาวน์โหลดหรือคัดลอกโฟลเดอร์โครงการไปไว้ที่เครื่องของคุณ (เช่น `C:\Users\Anant\Desktop\leavesystem`)
2. เปิด PowerShell หรือ Command Prompt เข้าไปที่โฟลเดอร์โครงการ:
   ```powershell
   cd C:\Users\Anant\Desktop\leavesystem
   ```
3. เริ่มรันเซิร์ฟเวอร์ด้วยคำสั่ง Python 3:
   ```powershell
   python server.py
   ```
4. เปิดเบราว์เซอร์ไปที่ลิงก์ใช้งาน:
   👉 **[http://localhost:8000](http://localhost:8000)**

## 🔑 บัญชีผู้ใช้งานสำหรับทดสอบระบบ (Test Accounts)

ฐานข้อมูลเริ่มต้น (Seed Data) ได้สร้างผู้ใช้งานที่มีการจำลองประวัติการลางานและเข้ารหัสผ่านด้วย PBKDF2 ไว้แล้วดังนี้ (**ทุกรหัสเข้าด้วยรหัสผ่านเดียวกันคือ: `password123`**):

| ชื่อผู้ใช้งาน (Username) | สิทธิ์เข้าใช้งาน (Role) | แผนก (Department) | ชื่อ-นามสกุลพนักงาน |
|---|---|---|---|
| `employee1` | พนักงาน (Employee) | IT | สมิตา อาสาคติ |
| `employee2` | พนักงาน (Employee) | Marketing | กิตติพงษ์ แก้ววิเชียร |
| `employee3` | พนักงาน (Employee) | Sales | นรีรัตน์ รักษาพล |
| `manager1` | หัวหน้างาน (Manager) | IT | วิชาญ ศรีสมบูรณ์ |
| `hr1` | HR / แอดมิน | HR | ประภาส เมืองดี |
